== Current State

typed-wasm applies TypeLL's 10-level type safety to WebAssembly linear memory.
Pre-alpha / research stage. Core insight: WASM linear memory is a schemaless
database; typed-wasm adds region schemas and typed access operations.

Killer feature: multi-module shared memory type safety across language boundaries
(e.g. Rust Module A shares WASM memory with ReScript Module B -- neither source
type system covers the boundary; typed-wasm does).

Key capabilities delivered:

* Idris2 ABI proofs (src/abi/) -- region schemas, typed access, 10-level verification
* Zig FFI (ffi/zig/) -- C-ABI bridge for runtime region management, typed load/store
* `.twasm` grammar (spec/grammar.ebnf) -- EBNF surface syntax
* 10-level spec (spec/10-levels-for-wasm.adoc) -- DB-to-WASM level mapping
* Example `.twasm` files demonstrating all features
* arXiv-ready whitepaper (docs/WHITEPAPER.md + docs/arxiv/typed-wasm.tex)
* No dangerous patterns (believe_me, assert_total, unsafeCoerce)
